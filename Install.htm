INSTALLATION or UPGRADE instructions for FusionTicket Beta 5

Requirements

The server must have these programs/settings to run Fusion Ticket:

  • PHP version =>5.2.1 through 5.2.10, tested with 5.2.7.  PHP <5.2.1 or >5.2.10 IS NOT SUPPORTED!
  • MySQL version >4.1 with MySQLi is Mandatory. Tested with version 5.0.51a
  • Enable email support in PHP - Check email security settings.
  • Adobe Acrobat Reader (to view/print PDF files)
  • cURL should be enabled for some features (PayPal, Authorize.net)
  • openssl should be enabled for some features i.e. https:
  • XML parser should be enabled
  • iconv support for non-latin text in PDF templates

Some third party code is already included with the distribution:

  • Smarty (see smarty.php.net) is installed in .../smarty folder
  • EzPDF in .../classes
  • HTMLMimeMail in .../classes
  • Fusion Ticket provides all required pear components in .../includes/pear folder.

Fusion Ticket has been tested on CentOS, Fedora Core, Ubuntu, and Windows with Apache 2.0. Generally, the latest stable version of software will have no problems but there are no guarantees.

If there are any problems with the above requirements, or the web host, contact mailto:hosting@fusionticket.co.uk. This e-mail address is being protected from spam bots. You need JavaScript enabled to view it for more information.

Fusion Ticket can provide hosting which is always up-to-date for security and compatibility. Use the same version that we design the software on. All we ask is that you donate regularly to the project. 

We do not recommend the Beta Versions of Fusion Ticket for Production Use!

Upgrade to Beta5

  • Backup your database!
  • Download the archive Fusion Ticket Beta5
  • Assuming the name of the Beta4 folder is /Fusionticket/ rename it to ../Fusionticket _B4/ (or whatever you like).
  • Create a new folder and give it the name of the original folder, in this case ../Fusionticket/
  • Copy the downloaded files to the new /Fusionticket/ folder.
  • Copy the the original /files/ folder to the new folder
  • Copy the original file init_config.php, now located in ../Fusionticket _B4/includes/config/ to the new folder.
  • Set the following folders and subfolders to writable 
	.../includes/config/*
	.../includes/fonts/*
	.../includes/tmp/*
	.../files/*
  • Browse to the installation script ../yourhost/Fusionticket/inst/index.php and select ‘Upgrade’
  • Remove the .../inst folder from your server
  • Reset the .../includes/config folder to Read-only.
  • Check the new .tpl files against any you may have previously modified.
  • Compare the new .../includes/config/init_config.php against the old one.

Full Installation

  • Download the archive Fusion Ticket and decompress it somewhere in the web root path of the web server:
  • This produces a directory called Fusionticket - you can rename it.
  • Create new database using the MySQL client, for example: mysqladmin create Fusionticket
  • Set the following folders and subfolders to writable 
	.../includes/config/*
	.../includes/fonts/*
	.../includes/tmp/* 
	.../files/*
  • Browse to the installation script /yourhost/Fusionticket/inst/index.php and select ‘Full Installation’
  • Remove the .../inst folder from your server
  • Reset the .../includes/config folder to Read-only.

Problems

Some known problems can occur due to various PHP configurations.
The solution to most of them is renaming _.htaccess file to .htaccess
Some web servers will not read .htaccess; in this case, edit the config file:

…/includes/config/init_common.php

Register globals
For security reasons this php flag should be off.

Install will produce a warning if it detects it is on.

If renaming _.htacess did not solve the problem then remove these two lines: 

echo "Set register_globals to OFF please.  Bye.”;
exit;

Blank screen
If you see a blank screen instead of the install page, your php is an older version and is not displaying the errors.

Update to a newer php version.

Headers already sent
This error displays because you have entered an extra whitespace in a file, usually in …/init_common.php.

Edit this file and remove everything before the <?php at the beginning and after ?> at the end.

Some editors will insert a blank line even if you do not, in this case try another editor.

Email Issues

Windows servers may not send email. If you experience this issue change the code in the email templates.

Standard email format: 

<FROM email="yourname<yourname@yourdomain.com"/>
<TO email="$user_firstname $user_lastname<$user_email>"/>

Windows email format: 

<FROM email="yourname@yourdomain.com" />
<TO email="$user_email"/>

Service is temporarily unavailable

This may be an installation bug.

Check the database table "ShopConfig". The field "status" must be set to ON.

Variables not found / Windows Apache

Some problems with "EasyPHP" have been noticed, missing variable errors are noticed sometimes.

We recommend you use "XAMPP" instead over "EasyPHP".
